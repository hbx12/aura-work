## What we do not support via GitHub Issues

- Individual API provider billing or quota problems
- Lost recovery keys (E2EE data cannot be recovered without the key)
- Modified or unofficial builds

## Getting help effectively

1. State Aura OS version (`Settings → About` or `npm run cli -- status`).
2. Include OS and architecture (macOS/Windows/Linux, x64/aarch64).
3. Attach relevant audit log excerpts — redact API keys and file paths you do not want public.

Maintainers respond on a best-effort basis. For security issues, use [SECURITY.md](./SECURITY.md) instead.
