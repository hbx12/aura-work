## Purpose

Support **20 languages** in the Aura Work UI with RTL layout for Arabic and Persian, and Weblate-compatible locale files.

## Key files

| File | Purpose |
|-------|---------|
| `packages/i18n/src/catalog.ts` | Source catalog + translations |
| `packages/i18n/locales/*.json` | Weblate flat JSON files |
| `packages/i18n/WEBLATE.md` | Weblate setup instructions |
| `apps/desktop/src-tauri/src/i18n.rs` | SQLite locale storage + system language detection |
| `apps/desktop/src/hooks/useI18n.ts` | React hook + `dir=rtl` for RTL languages |

## Supported languages (20)

English, Arabic, Spanish, French, German, Portuguese, Chinese (Simplified/Traditional), Japanese, Korean, Hindi, Indonesian, Turkish, Russian, Italian, Dutch, Polish, Vietnamese, Thai, Persian.

## Behavior

- Uses system language if supported → falls back to English
- User can override language from Settings
- `ar` and `fa` → `document.documentElement.dir = rtl`

## How to test

1. Settings → Language → العربية (or any language)
2. Verify RTL in nav rail for Arabic/Persian
3. `npm run build:locales -w @aura-os/i18n` → updates `locales/*.json`

## Weblate workflow

```bash
# Build locale files from source catalog
npm run build:locales -w @aura-os/i18n
```

Upload `packages/i18n/locales/` to Weblate as JSON flat key-value. Translations submitted via Weblate are synced back via PR.

## Adding a new language

1. Add translations to `packages/i18n/src/catalog.ts`
2. Run `npm run build:locales -w @aura-os/i18n`
3. Add locale key to `useI18n.ts` (if RTL, add to RTL list)
4. Add language entry in `Settings` page dropdown
5. Update this doc's language list
