Documentation

Everything you need to know about Aura Work — the open source, multi-provider desktop AI agent platform.

Overview

Aura Work is a free, open-source desktop AI agent platform built with Tauri 2 + React 19 and a Rust backend. It runs locally on your machine — no cloud dependency, no telemetry, no sign-up required.

The platform uses a multi-sidecar architecture: each major capability runs in its own lightweight sidecar process (agent, browser, VM, cloud sync, etc.), communicating over HTTP. This makes the system modular, crash-isolated, and easy to extend.

Key philosophy: local-first, permission-gated, extensible. You own your data, your keys stay on your machine, and the AI acts within boundaries you define.

Installation

Aura Work is in early alpha. Pre-built binaries for Windows, macOS, and Linux are available on the GitHub Releases page.

To build from source:

git clone https://github.com/hbx12/aura-work.git
cd aura-work
npm install
npm run tauri dev

Prerequisites: Node.js 20+, Rust, and Tauri 2 system deps.

Configuration

Configure providers, routing policies, and themes through the desktop app's Settings page or via the aura.jsonc file in your project root.

Supported configuration options include AI provider keys, default models, routing policies, theme choice, and scheduled task definitions.

Smart Routing

Route each task to the best model automatically based on your policy:

Quality-first

Best model for the job. Default policy.

Cost-first

Cheapest model that can handle the task.

Privacy-first

Prefer local models; redact secrets before sending to cloud.

Local-only

Ollama only. No cloud requests ever.

Manual

Use the specific model you pick per provider.

Routing can be set globally or per-project. Each decision is logged in the audit trail.

Permission System

High-impact actions require explicit approval. The permission engine categorises actions by risk and prompts the user before execution.

Built-in permission profiles:

Read-only files

Grant file read access to all paths.

Safe automation

Read + write files, shell read commands.

Research

Read files and browse the web.

Every permission request is recorded in the audit log with timestamps, actor info, and decision outcomes.

Encrypted Vault

API keys and secrets are stored in an encrypted vault backed by the operating system's secure storage (Keychain on macOS, Credential Manager on Windows, Secret Service on Linux).

The vault is device-bound and supports zero-trust recovery keys when cloud sync is enabled.

Scheduling

Schedule recurring AI tasks — daily stand-ups, weekly reports, hourly monitoring. Each task runs with its own permission profile and routing policy.

Supported cadences: manual, hourly, daily, weekly, weekdays, or custom cron expressions.

Custom Tools

Write your own TypeScript tools and run them inside the agent. Tools are sandboxed, can be tested live, and support JSON-schema argument validation.

Templates available: API Fetch (fetch external APIs) and File Utils (read/write/check files). You can install tools globally or per-project.

Example: a weather fetch tool in ~20 lines of TypeScript.

MCP Servers

Aura Work fully supports the Model Context Protocol (MCP). Connect any MCP-compatible server via stdio or SSE transport. Servers can be installed from the marketplace or configured manually.

MCP servers appear as tools the agent can call — extending the agent's capabilities with databases, APIs, file systems, and more.

Plugin System

Plugins are packaged extensions that can add tools, sidecars, and UI components. Installed via the Plugins page in the desktop app. The Chrome extension (aura-chrome) is an example that bridges the browser with the agent.

Sidecar Architecture

Aura Work runs 8 sidecar processes, each responsible for a specific capability. Sidecars are independent processes that communicate via HTTP — if one crashes, the rest keep running.

Agent

Core AI agent orchestration and task execution.

Bridge

Connects browser extension (Chrome) and Office add-ins (Word, Excel, PowerPoint).

Browser Helper

Manages browser automation profiles and sessions.

Cloud Sync

End-to-end encrypted sync across devices.

Computer Use

Lets the agent control desktop applications (experimental).

Plugins Helper

Manages plugin installation, MCP servers, and custom tools.

VM Helper

Provides isolated virtual machine execution.

Themes

Aura Work ships with 17 hand-crafted themes covering light, dark, and AMOLED modes. Switch between Nord, Dracula, Cyberpunk, Sakura, Monokai, Tokyo Night, Catppuccin, Solarized, Gruvbox, and more.

Custom theme creation is supported through the theme configuration file.

CLI

Aura Work includes a command-line interface (aura-cli) for headless operation — run tasks from the terminal, integrate with CI/CD pipelines, or script your workflows.

aura run "Review this PR for security issues"
aura schedule --daily "Summarise my GitHub notifications"
