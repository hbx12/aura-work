الأذونات

مستويات أذونات تتحكم في ما يمكن للوكلاء فعله

يستخدم Aura Work نظام أذونات متدرج مع 3 مستويات مدمجة. كل مستوى يمنح وصولاً متزايداً لموارد النظام والأدوات.

مستويات الأذونات

قراءة فقط

قراءة ملفات، بحث في الكود، تصفح الويب. لا تعديلات على نظام الملفات ولا تنفيذ كود.

أتمتة آمنة

قراءة وكتابة ملفات، تنفيذ أوامر مصرح بها، وصول للشبكة. تنفيذ معزول مع حدود موارد وتسجيل تدقيق.

بحث

وصول كامل للنظام يشمل نظام الملفات والشبكة وتنفيذ الكود وأتمتة المتصفح. كل الإجراءات مسجلة وقابلة للتراجع.

Customization

إنشاء مستوى مخصص

حدد مستويات مخصصة في settings.json بدمج أذونات فردية. كل إذن يمنح وصولاً لقدرة محددة.

{
  "profiles": [
    {
      "id": "my-custom-profile",
      "permissions": ["files.read", "files.write", "network.http"],
      "extends": "safe-automation"
    }
  ]
}
نموذج الأذونات

فهم نموذج الأذونات

يستخدم Aura Work نظام أذونات خماسي المستويات للتحكم الكامل في ما يمكن للذكاء الاصطناعي فعله:

المستوى 1: قراءة نظام الملفات

السماح بقراءة الملفات والمجلدات. هذا هو المستوى الأساسي المطلوب لمعظم المهام. يمكن تقييده بمجلدات محددة.

المستوى 2: كتابة نظام الملفات

السماح بإنشاء وتعديل وحذف الملفات. مطلوب لمهام البرمجة. يمكن تقييده بمسارات محددة.

المستوى 3: تنفيذ الأوامر

السماح بتشغيل أوامر shell. مطلوب لتشغيل الاختبارات والبناء. يمكن تقييده بأوامر مسموحة فقط.

المستوى 4: الوصول للشبكة

السماح بالاتصال بالإنترنت. مطلوب لاستدعاءات API والبحث على الويب. يمكن تقييده بنطاقات محددة.

المستوى 5: التحكم الكامل

وصول غير مقيد. يتطلب تأكيدا صريحا من المستخدم. غير موصى به للاستخدام اليومي.

يمكن تكوين الأذونات بشكل دقيق في ملف aura.config.json. كل طلب يعرض الأذونات المطلوبة قبل التنفيذ.
