Quick Start

From zero to running your first AI agent task.

1. Install

Download the latest release from GitHub Releases. Installers are available for Windows, macOS, and Linux.

2. Launch

Open Aura Work. The app starts with a default project and the Agent panel ready. You'll see the chat interface where you can type your first task.

3. Configure a provider

Go to Settings → Providers. Add an API key for any provider (OpenAI, Anthropic, etc.) or start Ollama locally for free. Each provider is validated automatically.

4. Run your first task

Type a prompt like: "Create a new React component that fetches data from an API". The agent plans the task, executes it step by step, and shows you the results.

5. Explore

Try skills from the registry, install MCP servers, configure routing policies, set up scheduled tasks, or pair the CLI for terminal-based control.

System Requirements

Installation details

Aura Work runs on all major desktop platforms. Below are the specific requirements and install methods for each operating system.

Download options

GitHub Releases: Download the latest installer from github.com/hbx12/aura-work/releases
Windows (winget): winget install aura-work
macOS (Homebrew): brew install aura-work
Linux (AppImage): Download Aura-Work-x86_64.AppImage and run chmod +x Aura-Work-x86_64.AppImage && ./Aura-Work-x86_64.AppImage

Verifying installation

Open a terminal and run:

aura version
# Output: Aura Work 0.1.0-alpha.1 (build abc1234)

Directory structure

After installation, Aura Work creates the following directory structure:

~/.aura/                  # Configuration directory
 config.json              # Main configuration file
 vault/                   # Encrypted credential storage
 logs/                    # Application logs
 projects/                # User projects
 registry/                # Skills and sidecar registry
   skills/                # Skill manifest files (.agents)
   sidecars/              # Sidecar configuration

First Task

Your first task in detail

Step 1: Launch the application

Open Aura Work. The welcome screen displays the agent chat panel. Select your preferred provider from the dropdown in the top bar. If you set environment variables, the provider is automatically detected and configured.

Step 2: Type your first prompt

In the chat input, type a task such as:

"Create a new React component that fetches user data from an API and displays it in a table"

Step 3: Agent execution cycle

The agent processes your request in three phases:
Plan — The agent analyzes your request and creates a step-by-step execution plan. Tools and skills are selected based on the task requirements.
Execute — Each step is executed sequentially. The agent calls tools, reads files, writes code, and verifies results at each stage.
Deliver — The final result is presented in the chat panel with file contents, explanations, and next-step suggestions.

Step 4: Review results

The agent displays the created component code, explains the structure, and offers to refine it based on your feedback. You can iterate by typing follow-up instructions such as "Add loading states and error handling" or "Convert to TypeScript".

Troubleshooting

Common issues

No provider configured

Symptom: The agent responds with "No provider available."
Fix: Set at least one API key as an environment variable or configure a provider in Settings → Providers. For local models, start Ollama with ollama serve.

Bridge connection refused

Symptom: The CLI reports "Bridge not reachable."
Fix: Ensure the desktop app is running. Check that the Bridge sidecar is enabled in Settings → Sidecars. Restart the application if the issue persists.

Application logs

Logs are located at ~/.aura/logs/. The most recent log file is aura.log. Enable debug logging by setting AURA_LOG_LEVEL=debug as an environment variable.

Resetting configuration

To reset Aura Work to its default state, quit the application and delete ~/.aura/config.json. Restart the app to generate a fresh configuration. Your projects and skills registry are preserved.
