بداية سريعة

من الصفر إلى تشغيل أول مهمة ذكاء اصطناعي.

هذا الدليل سيأخذك من التثبيت إلى أول مهمة مدعومة بالذكاء الاصطناعي في أقل من 5 دقائق. Aura Work متوفر لأنظمة Windows و macOS و Linux.

المتطلبات الأساسية

المنصةالمتطلبات

1. التثبيت

حمّل أحدث إصدار من إصدارات GitHub. برامج التثبيت متوفرة لـ Windows (.msi) و macOS (.dmg) و Linux (.deb، .AppImage).

Windows: شغّل ملف .msi واتبع التعليمات. سيُضاف التطبيق إلى قائمة ابدأ.

macOS: افتح ملف .dmg واسحب Aura Work إلى مجلد Applications.

Linux: ثبّت حزمة .deb عبر sudo dpkg -i aura-work.deb أو شغّل ملف .AppImage مباشرة.

2. التشغيل

افتح Aura Work. يبدأ التطبيق بمشروع افتراضي ولوحة الوكيل جاهزة. سترى واجهة المحادثة حيث يمكنك كتابة أول مهمة.

عند التشغيل الأول، سيقوم التطبيق بـ:

  • إنشاء قاعدة بيانات SQLite محلية للمشاريع والمهام
  • تهيئة الخزنة المشفرة لمفاتيح API
  • تشغيل الخدمة المساعدة Agent (المنفذ 47821)
  • اكتشاف لغة نظامك وتطبيق الثيم المناسب

3. تكوين مزوّد

اذهب إلى الإعدادات ← المزوّدون. تحتاج إلى مزوّد ذكاء اصطناعي واحد على الأقل لاستخدام Aura Work. الخيارات:

الخيار أ: مزوّد سحابي (موصى به للبدء)

  1. 1. اضغط على مزوّد (OpenAI، Anthropic، Gemini، إلخ)
  2. 2. أدخل مفتاح API الخاص بك
  3. 3. اضغط "تحقق" لاختبار الاتصال
  4. 4. اختر النماذج التي تريد استخدامها

الخيار ب: مزوّد محلي (مجاني، دون حاجة لمفتاح API)

  1. 1. ثبّت Ollama على جهازك
  2. 2. اسحب نموذجاً: ollama pull llama3
  3. 3. في Aura Work، فعّل مزوّد Ollama
  4. 4. يكتشف التطبيق النماذج المتاحة تلقائياً

المزوّدون المحليون يبقون كل البيانات على جهازك — بدون مفاتيح API، وبدون حاجة للإنترنت.

4. تشغيل أول مهمة

اكتب طلباً في واجهة المحادثة. إليك بعض الأمثلة لتجربتها:

# مهام برمجية
"أنشئ مكوّن React جديد يجلب بيانات من واجهة API"
"أصلح الخلل في auth.ts حيث يفشل تسجيل الدخول مع الرموز الخاصة"
"أضف اختبارات وحدة لصنف UserService"

# مهام مستندات
"أنشئ تقرير PDF يلخص بنية المشروع"
"أنشئ جدول بيانات بنتائج الاختبارات"

# مهام بحث
"ابحث عن أحدث أفضل ممارسات أمان Node.js"
"قارن بين PostgreSQL و MongoDB لحالتنا الاستخدامية"

# مهام متصفح
"تصفح https://example.com واستخرج كل العناوين"
"املأ نموذج التواصل على الموقع"

سيقوم الوكيل بـ:

  1. 1. تحليل طلبك وإنشاء خطة
  2. 2. اختيار أفضل مزوّد ونموذج (حسب سياسة التوجيه لديك)
  3. 3. تنفيذ كل خطوة، مستدعياً الأدوات عند الحاجة
  4. 4. عرض النتائج عليك وطلب الموافقة على الإجراءات عالية التأثير

5. الاستكشاف

الآن بعد أن أصبحت أول مهمة تعمل، استكشف هذه الميزات:

  • المهارات — جرّب "استخدم مهارة البحث لـ..." أو "أنشئ جدول بيانات بـ..."
  • خوادم MCP — ثبّت خوادم MCP من لوحة الإضافات لقدرات إضافية
  • سياسات التوجيه — غيّر سياسة التوجيه من الإعدادات ← التوجيه
  • المهام المجدولة — أنشئ مهاماً متكررة من صفحة المهام المجدولة
  • واجهة الأوامر — ثبّت CLI للتحكم من الطرفية: npm install -g @aura-work/cli
  • أتمتة المتصفح — جرّب "تصفح هذا الموقع و..." لمهام الويب
  • التحكم بالحاسوب — فعّل ميزة التحكم بالحاسوب التجريبية لأتمتة سطح المكتب

الإعدادات الأساسية

🔀 سياسة التوجيه

اضبط كيفية اختيار الوكيل للمزوّدين من الإعدادات ← التوجيه:

  • الجودة أولاً — يستخدم أفضل نموذج متاح (موصى به للمهام المهمة)
  • التكلفة أولاً — يستخدم أرخص نموذج قادر على التعامل مع المهمة
  • الخصوصية أولاً — يستخدم المزوّدين المحليين فقط (Ollama، LM Studio)
  • محلي فقط — مثل الخصوصية أولاً، لكن بفرض أكثر صرامة
  • يدوي — أنت تختار المزوّد لكل مهمة

🔐 الأذونات

اضبط ما يمكن للوكيل فعله من الإعدادات ← الأذونات:

  • اسأل أولاً (افتراضي) — يسأل الوكيل قبل كل إجراء عالي التأثير
  • قراءة فقط — يمكن للوكيل فقط قراءة الملفات، لا تعديلات
  • وصول كامل — يمكن للوكيل فعل أي شيء (استخدم بحذر)

يمكنك أيضاً ضبط أذونات لكل فئة (ملفات، shell، متصفح، إلخ) وإنشاء ملفات تعريف مخصصة.

🎨 الثيم

اختر من بين أكثر من 35 ثيماً من الإعدادات ← عام ← الثيم. تشمل الخيارات الفاتح والداكن وAMOLED وثيمات فاخرة. ثيم system يطابق تلقائياً تفضيل نظام التشغيل لديك.

التثبيت المتقدم

خيارات التثبيت الأخرى

باستخدام Homebrew (macOS و Linux)

brew install aura-work

باستخدام Winget (Windows)

winget install aura-work

البناء من المصدر

git clone https://github.com/hbx12/aura-work.git
cd aura-work
npm install
npm run build
npm link

يتطلب Node.js 18+. يدعم Windows و macOS و Linux مع حزم منفصلة لكل منصة.
